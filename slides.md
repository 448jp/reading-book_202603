::title::

プレゼンテーションの目的

::content::

- プレゼンテーションの目的は、相手に **何かを伝える** こと
- **伝わらないと失敗** である
- プレゼンテーションとデモンストレーションは基本的に同じ

::title::

何を伝えるのか？（5W1H）

::content::

1. **誰に** 伝えるのか？
2. **いつ** 伝えるのか？
3. **どこで** 伝えるのか？
4. **なぜ** 伝えなければならないのか？
5. **誰が** 伝えるのか？
6. **どのようにして** 伝えるのか？

プレゼンテーションの種類

::content::

| 種類 | 特徴 |
|------|------|
| オーソドックス型 | 書かれていることを説明する |
| ビジー型 | ポスタープレゼンテーション |

| 完全比較型 | 競合や既存手法との比較で伝える |
| スティーブ・ジョブズ型 | ストーリーと演出で魅了する |

::title::

プレゼンテーションの視点

::content::

基本的に **顧客視点** で統一する

| 視点 | 例文 |
|------|------|
| 自分視点 | 「○○のクラウド版を **出荷します** 」 |
| 神様視点 | 「○○のクラウド版が **出荷されました** 」 |
| **顧客視点** | 「クラウドで○○を **手に入れることができます** 」 |

- 自分視点 → 発信者都合の表現
- 神様視点 → 第三者的で距離がある
- **顧客視点** → 聞き手が主語になり、自分事として受け取れる

プレゼンはドラマ

::content::

- **実在しないもの** 、有名でないものを比喩にしてはならない

::title::

テクニック：表現と演出

::content::

- **大きさ** を説明するときは身振りで説明する
- 実数を想像できる言葉で説明する
  - 「9月」ではなく、「**松茸を召し上がる頃には**」
- 問題提起のあとに **ホラーストーリー** （不安を煽る）を提示する
- 変化は **左から右** 、そして絶対に **右肩上がり** にする
- **第一声は大きな声で、挨拶を**
- 1分間に **数字を3つ** 織り交ぜると説得力が増す

::title::

テクニック：構成とスライド

::content::

- **他人の話、過去の話** の引用は効果がある
- デモンストレーションが終わったら **補足するスライドを見せる**
- 次に表示するスライドのことを話してから **スライドを切り替える** （Lynda方式）
- この資料を **多くの人に見せたい** 、また **喋りたい** と思うように作る

::title::

まとめ

::content::

1. プレゼンの目的は **伝えること** 。伝わらなければ失敗
2. 事前に **5W1H** を整理してから臨む
3. 視点は **顧客視点** で統一する
4. 美しい言葉・身振り・数字・Lynda方式など **テクニック** で説得力を高める
